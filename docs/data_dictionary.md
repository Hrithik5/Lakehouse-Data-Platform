# 📖 Data Dictionary

## Overview

The Gold layer contains curated, business-ready datasets generated from the Silver layer. These tables are optimized for analytical workloads and are registered in the AWS Glue Data Catalog, making them queryable through Amazon Athena.

---

# customer_summary

## Description

Provides customer-level aggregated metrics used for customer analytics, segmentation, and lifetime value analysis.

## Primary Key

| Column |
|---------|
| customer_id |

## Columns

| Column | Data Type | Description |
|---------|-----------|-------------|
| customer_id | STRING | Unique identifier of the customer. |
| customer_city | STRING | Customer's city. |
| customer_state | STRING | Customer's state. |
| total_orders | INTEGER | Total number of orders placed by the customer. |
| total_spent | DOUBLE | Total amount spent across all completed orders. |
| average_order_value | DOUBLE | Average value per order. |
| first_purchase | DATE | Date of the customer's first purchase. |
| last_purchase | DATE | Date of the customer's most recent purchase. |

## Business Use Cases

- Customer Lifetime Value (CLV)
- Customer Segmentation
- Regional Customer Analysis
- Purchasing Behavior Analysis

# Data Lineage

The Gold layer is generated through a Medallion Architecture.

```text
Raw CSV Files
        │
        ▼
 Bronze Layer
 (Raw Delta Tables)
        │
        ▼
 Silver Layer
 (Cleaned & Standardized)
        │
        ▼
 Gold Layer
 (Business Summary Tables)
```

---

# Data Quality Rules

The following quality checks are applied during the ETL process before generating the Gold layer.

### Bronze Layer

- Raw data ingestion
- Schema inference
- Delta format conversion

### Silver Layer

- Duplicate removal
- Null value handling
- Data type validation
- Text standardization
- Business transformations

### Gold Layer

- Business aggregations
- KPI generation
- Analytical summary tables
- Athena-ready datasets

---

# Gold Layer Relationships

Although the Gold layer consists of denormalized summary tables, the following logical relationships exist.

| Parent Entity | Related Table |
|---------------|---------------|
| Customer | customer_summary |
| Seller | seller_summary |
| Product | product_summary |
| Order | payment_summary |

These tables are designed for analytical workloads and minimize the need for complex joins during reporting.
